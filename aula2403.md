# 1 - cenário

"Um dono de uma cafeteria pequena chamado João quer um site para divulga seu site"

# 2 - briefing

## Qual o objetivo do site?

atrair clientes para a sua cafeteria e tambem fazer vendas online

## Qual é o público alvo?

Jovens na faixa etária de 18 a 35 anos

## O que o site precissa ter? 

-Uma Pagina inicial atrativa mas não confusa
-Cardápio com fotos e preços
-Horario de funcionamento
-Redes sociais e contato

# Funcionalidade - RF

1. Exibir página inicial atrativa com destaque para promoções
2. Apresentar cardápio online com fotos e preços
3. Permitir vendas online
4. Disponibilizar informações de contato (telefone e WhatsApp)
5. Sistema de gerenciamento simples para João atualizar produtos e preços

# Usabilidade - RNF

1. Interface simples, moderna e intuitiva
2. Navegação rápida com poucos cliques
3. Site responsivo
4. Uso de imagens de alta qualidade

# Confiabilidade - RNF

1. O sistema deve funcionar 24/7
2. Carrinho de compras não pode perder dados
3. Sistema deve lidar bem com múltiplos acessos simultâneos

# Desempenho - RNF

1. Tempo de carregamento rápido +/- 3seg
2. Suporte a acessos simultâneos sem lag

# Suportabilidade - RNF

1. Código organizado e fácil de manter
2. Compatível com diferentes navegadores

# Estrutura (mapa do site)
- Página inicial
- Cardápio
- Produto 
- Carrinho
- Checkout (finalização da compra)
- Contato
- Avaliações
